The `connection` generator wires a <Link path="guides/ts-smithy-api">Smithy API</Link> to a <Link path="guides/ts-opensearch">TypeScript OpenSearch</Link> project, configuring local development so both start together automatically.

## Prerequisites

Before using this generator, ensure you have:

1. A <Link path="guides/ts-smithy-api">Smithy TypeScript API</Link> project (generated with `ts#api` using `--framework=smithy`)
2. A <Link path="guides/ts-opensearch">`ts#opensearch`</Link> project

## Generator Output

The generator updates your Smithy API's `project.json` to add a dependency from its `dev` target to the OpenSearch project's `dev` target. No source files are modified.

## Using OpenSearch in Operations

Import the client and index schema from the OpenSearch package and use them inside your operation implementations:

```ts title="packages/api/src/operations/search-examples.ts"
import { getOpenSearchClient, exampleIndexName } from ':my-scope/my-search';
import {
  SearchExamplesOperationInput,
  SearchExamplesOperationOutput,
} from '../generated/ssdk/index.js';
import { ServiceContext } from '../context.js';

export const searchExamples = async (
  input: SearchExamplesOperationInput,
  _ctx: ServiceContext,
): Promise<SearchExamplesOperationOutput> => {
  const client = await getOpenSearchClient();
  const { body } = await client.search({
    index: exampleIndexName,
    body: { query: { match: { title: input.query } } },
  });
  return { items: body.hits.hits.map((hit: any) => hit._source) };
};
```
