title: Starting Local OpenSearch

The generator configures a `dev` target that starts a classic (non-serverless) [OpenSearch](https://opensearch.org/) container — used because OpenSearch Serverless has no local emulator, but the index/document APIs are compatible — and syncs your indices against it. Use the project's `dev` target:

<NxCommands commands={['dev <project-name>']} />

This automatically:
1. Pulls the OpenSearch image (`pull-image` target)
2. Starts a single-node container with the security plugin disabled (no auth needed locally)
3. Creates/updates indices from the zod schemas under `src/indices/`

Multiple `ts#opensearch` projects in the same workspace share a single local container. Stopping `dev` (e.g. with `Ctrl+C`) removes the container, but preserves the named volume so your data persists across restarts.
